关于这个项目

这是我的第一个 CI/CD 实践项目。一个纯静态网站，没有任何后端， 只有 HTML、CSS 和 JS。但它的部署流程是全自动的：

1
写代码，push 到 GitHub
2
GitHub Actions 自动触发
3
构建并部署到 GitHub Pages
4
网站自动上线

CI/CD 流程

下面的流程展示了每次 git push 后发生的事情。 整个过程由 .github/workflows/deploy.yml 定义。

触发
push to main
→
Build Job
checkout → configure → upload
→
Deploy Job
deploy-pages → 网站上线
关键概念：
  • permissions — 声明工作流需要哪些权限（读代码、写 Pages）
  • concurrency — 同一时间只允许一个部署运行，避免冲突
  • needs — deploy 依赖 build，构建成功后才部署
  • artifact — 构建产物的临时存储，跨 job 传递文件
